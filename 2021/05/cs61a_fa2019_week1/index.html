CS61A课程2019年秋季学年网课学习笔记

Python

课程一开始就有一个理念吸引到了我，程序设计语言只是我们去抽象问题、分析抽象问题的途径，并不是我们学习的根本目的。

经由Python学习这些内容，应该也是出于这也考虑吧，Structure and Interpretation of Computer Science的学习注定是抽象的，思维的狂欢，那么为了这一目的选择了便于理解的python，也就是可以理解的了。

Debugging_principles

  • 渐进地编写程序并进行调试
  • 逐步分理处问题所在位置
  • 再次确认自己的预计想法是否有问题，有时候你觉得程序中某个函数应该有的功能其实是与实际情况不相符的
  • 向dl咨询——私以为这个是最有用的

Environment

由于python是解释型语言，可以一行一行代码边解释边执行，所以每一句python语句的上下文显得格外重要，如果没有语境的配合，甚至无法确定函数名是否还是原来的意义。

将名绑定(bind)到一个值上，然后通过这个名来检索它对应的值，这个过程意味着python解释器需要维护一些内容，在这些内存中，解释器保存着这些对应关系，通过这些关系可以跟踪名与值的绑定，这种内存被称作环境。

Python3 Library Docs

python3的文档库

Function

pure_function

不产生任何副作用，只有输入输出的函数成为pure_function。

此处的副作用可以是输入输出，可以是修改内存内容，或者是其他一些除了计算并且返回值之外的东西。

以abs(number):为例，这个函数不会产生其他的影响，只会对传入的number值求绝对值然后返回绝对值。

non_pure_func

与上问的pure_function相对，此处以print()函数为例，print函数既返回了固定的None值，还想屏幕输出了内容。

此处需要注意的是，由于print函数有固定的返回值，所以在调用时需要谨慎。
